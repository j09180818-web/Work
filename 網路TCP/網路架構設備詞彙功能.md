IPv4 總共 32 bit而由此可以計算出我們有幾個Bit可以用
可用 Host 數 = 2^n - 2其中n = Host Bit（Host 位元數）
-2的原因是因為以下兩個位子必須占用
在一般 IPv4 子網路中：
第一個 IP（最小的 IP） → Network Address（網路位址）
最後一個 IP（最大的 IP） → Broadcast Address（廣播位址）
中間所有 IP → 可分配給 Host 使用

1. 子網路遮罩中，被遮住的地方是網段位。
   例如 255.255.0.0，代表前兩組是網段號碼 X.X。

2. 沒被遮住的地方是 Host 位。
   例如 192.168.0.0/16，可用 Host 是 192.168.0.1 ~ 192.168.255.254。

3. 如果遮罩出現 128、192、224、240、248、252，
   代表遮罩切在某一組中間，需要算跳號。

4. 跳號公式：
   跳號 = 256 - 遮罩數字

5. 每一個網段的第一個 IP 是 Network Address，
   每一個網段的最後一個 IP 是 Broadcast Address，
   中間才是可用 Host。

   # 子網路遮罩計算小筆記

## 1. 子網路遮罩的作用

子網路遮罩 Subnet Mask 用來決定：

```text
哪一部分是網段 Network
哪一部分是主機 Host
```

簡單記：

```text
255 的地方 = 網段位，固定
0 的地方 = Host 位，可變
128 / 192 / 224 / 240 / 248 / 252 = 遮到一半，要算跳號
```

---

## 2. 常見遮罩對照

| 子網路遮罩           | CIDR | 網段位       | Host 位    |  每段可用 Host |
| --------------- | ---: | --------- | --------- | ---------: |
| 255.0.0.0       |   /8 | 第 1 組     | 第 2、3、4 組 | 16,777,214 |
| 255.255.0.0     |  /16 | 第 1、2 組   | 第 3、4 組   |     65,534 |
| 255.255.255.0   |  /24 | 第 1、2、3 組 | 第 4 組     |        254 |
| 255.255.255.128 |  /25 | 前 25 bit  | 後 7 bit   |        126 |
| 255.255.255.192 |  /26 | 前 26 bit  | 後 6 bit   |         62 |

---

## 3. 如果遮罩切到一半，要算跳號

只要看到：

```text
128、192、224、240、248、252
```

就要算跳號。

公式：

```text
跳號 = 256 - 遮罩數字
```

範例：

```text
255.255.255.192
跳號 = 256 - 192 = 64
```

所以最後一組會切成：

```text
0 ~ 63
64 ~ 127
128 ~ 191
192 ~ 255
```

---

## 4. 每一個網段的規則

每一個網段都有：

```text
第一個 IP = Network Address 網路位址，不能給設備用
中間 IP = 可用 Host IP
最後一個 IP = Broadcast Address 廣播位址，不能給設備用
```

---

## 5. 例題

題目：

```text
IP：192.168.10.70
Mask：255.255.255.192
```

### 步驟 1：算 CIDR

```text
255.255.255.192 = /26
```

因為：

```text
255 = 8 bit
255 = 8 bit
255 = 8 bit
192 = 2 bit

8 + 8 + 8 + 2 = 26
```

### 步驟 2：算跳號

```text
跳號 = 256 - 192
跳號 = 64
```

### 步驟 3：列出網段

```text
第 1 段：192.168.10.0   ~ 192.168.10.63
第 2 段：192.168.10.64  ~ 192.168.10.127
第 3 段：192.168.10.128 ~ 192.168.10.191
第 4 段：192.168.10.192 ~ 192.168.10.255
```

### 步驟 4：判斷 IP 落在哪段

```text
IP：192.168.10.70

看最後一組 70
70 落在 64 ~ 127
所以它在第 2 段
```

---

## 6. 例題答案

```text
IP：192.168.10.70
Mask：255.255.255.192
CIDR：/26
跳號：64

所屬網段：
192.168.10.64 ~ 192.168.10.127

Network Address：
192.168.10.64

可用 Host：
192.168.10.65 ~ 192.168.10.126

Broadcast Address：
192.168.10.127
```

---

## 7. 最重要記法

```text
先看 Mask
再算跳號
再找 IP 落在哪一段

每段第一個是 Network
每段最後一個是 Broadcast
中間才是 Host
```

一句話：

```text
遮罩越多，網段切越細，每個網段可用 Host 越少。
```

# MAC :MAC 層
媒體存取控制層
在 TCP/IP 或 OSI 架構中，MAC 層屬於：
OSI 第 2 層：資料鏈結層 

資料鏈結層 Data Link Layer又可以再細分成兩個部分：
├─ LLC 層：Logical Link Control
└─ MAC 層：Media Access Control
在同一個 LAN 裡，資料要送給哪一台設備，以及大家如何輪流使用網路線或無線電波。
此層功能有
1.指定區域網路內的收件人
IP 位址通常用來判斷「目標在哪個網路」。
MAC 位址則是用來判斷「在這個區域網路內，資料要交給哪一台設備」。
在傳遞網路資料時封包會如下
Ethernet Frame乙太網路
├─ 目的 MAC：PLC 的 MAC Address
├─ 來源 MAC：HMI 的 MAC Address
├─ 資料內容：IP Packet / TCP / Modbus TCP
└─ 檢查碼

而在LAN中主要看的就是MAC ADDRESS
在以前使用 Hub 或半雙工網路時，多台設備共用同一條傳輸媒介，可能會發生碰撞。
MAC層就會決定什麼時候送如果碰撞後如何解解決
不過現代 Switch 網路多半是全雙工 Full-Duplex，碰撞問題已經很少見。
1. MAC 層本身不負責管 IP。
2. MAC 層負責 LAN 內 Ethernet Frame 的來源 MAC / 目的 MAC。
3. IP 層才負責來源 IP / 目的 IP / 子網路 / Gateway / 路由。
4. 同網段時，目的 MAC 是目標設備。
5. 跨網段時，目的 MAC 是 Gateway。
6. IP 目的地可以是遠端設備，但 MAC 目的地永遠只是目前這一段 LAN 的下一站。
# MAC ADDRESS
MAC Address / 實體位址 / 硬體位址 是網路卡的硬體識別碼
MAC Address 通常是 48 bits，也就是 6 Bytes。
00-1A-2B-3C-4D-5E
拆分成
00-1A-2B    -    3C-4D-5E
廠商代碼          廠商自行分配編號
OUI              NIC Specific
1. MAC Address 理論上設計為全球唯一。
2. MAC Address 通常由廠商 OUI + 廠商分配編號組成。
3. 48-bit MAC Address 數量非常大。
4. 正常出廠設備不應該重複。
5. 但人為修改、VM 複製、廠商燒錄錯誤，可能造成重複。
6. 只要在同一個 LAN / VLAN 內 MAC 重複，就可能造成通訊異常。
7. MAC 重複常見現象是 Switch MAC Table 跳動，也就是 MAC Flapping。

IP Address / Internet Protocol Address
中文常叫：

這台設備在哪個網路？
資料要送到哪個目的地？
需要直接送，還是交給 Gateway？

因此總結

# Switch交換器 VS Hub集線器
| 比較項目                       | Hub                     | Switch                     |
| -------------------------- | ----------------------- | -------------------------- |
| **OSI層**                   | 第1層（Physical Layer，實體層） | 第2層（Data Link Layer，資料鏈結層） |
| **判斷MAC位址**                | ❌ 不會                    | ✅ 會                        |
| **是否有MAC Table**           | ❌ 沒有                    | ✅ 有（MAC Address Table）     |
| **是否學習MAC**                | ❌ 不會                    | ✅ 會（MAC Learning）          |
| **封包轉送方式**                 | 收到就全部送                  | 查MAC Table，只送目的Port        |
| **Unicast（單播）**            | 全部Port都收到               | 只有目的Port收到                 |
| **Broadcast（廣播）**          | 全部送                     | 全部送（必要時）                   |
| **Unknown Unicast（未知MAC）** | 全部送                     | 全部送（第一次找不到MAC時）            |
| **Collision Domain（碰撞網域）** | 全部Port共用一個              | 每個Port各自獨立                 |
| **Broadcast Domain（廣播網域）** | 一個                      | 一個（未切VLAN時）                |
| **頻寬（Bandwidth）**          | 全部設備共享                  | 每個Port獨立                   |
| **雙工模式（Duplex）**           | Half Duplex（半雙工）        | Full Duplex（全雙工）           |
| **碰撞（Collision）**          | 容易發生                    | 幾乎沒有                       |
| **傳輸效率**                   | 低                       | 高                          |
| **安全性**                    | 差（所有設備都收到封包）            | 較佳（只有目的Port收到）             |
| **價格**                     | 較便宜（現在已很少使用）            | 較貴（目前主流）                   |
| **目前是否常見**                 | 幾乎淘汰                    | 幾乎所有企業與家庭都使用               |


# Router（路由器）
Router 主要功能：跨網段轉送 IP 封包。
它不一定包含 DNS，但現代家用路由器、工業路由器常常會附加 DNS Proxy / DNS Forwarder。
它不等於 Switch，但有些設備會把 Router + Switch 做在同一台裡，例如家用分享器。
它常常是 Gateway，因為 Gateway 是用戶端設定的「離開本網段出口」，而這個出口通常就是 Router 的某個 IP。
1. 要離開本網段時，封包通常交給 Router。
2. 終端設備設定的 Gateway，通常就是 Router 在同網段上的 IP。
3. Gateway 不是一種獨立設備名稱，而是一個角色 / 設定值。
4. Router、L3 Switch、防火牆都可以提供 Gateway 功能。


#  Port Forwarding（連接埠轉發 / 通訊埠對映）。
NAT 是讓 LAN 內的設備對外連線時，Router 將 Private IP 轉換成 Public IP。
而Port Forwarding 則是在 Router 建立一張轉送規則表，告訴 Router：當外部有人連到某個 Port 時，要把封包轉送到 LAN 中指定的設備。
外部設備連到 Router 的某個 Port。
「我就在路由器的表單上設定。」
========================
Port Forwarding
========================

Port Forwarding 是 Router 的一項功能。

用途：
在 Router 建立一張 Port 對照表（轉送規則）。

作用：
當 Internet 有設備連到 Router 的 Public IP + Port 時，
Router 會依照設定的規則，
將封包轉送到 LAN 中指定的設備。

例如：

PLC
192.168.1.20
Port：502

Router 設定：

Public IP:502
        ↓
192.168.1.20:502

之後只要有人連：

114.35.100.50:502

Router 就會查詢 Port Forwarding 規則，
並將封包轉送給 PLC。

========================
一句話記住
========================

NAT：
讓 LAN 的設備可以對外連線。

Port Forwarding：
讓 Internet 可以依照指定 Port，
連線到 LAN 中指定的設備。
如果沒有設定就會直接丟棄無視這個封包
所以它的名字才叫做 Port Forwarding（連接埠轉發）。這個名稱本身就已經描述了它的工作內容。

# LAN
LAN（Local Area Network，區域網路）內，資料鏈結層（Data Link Layer）是使用 MAC Address 來傳送 Ethernet Frame。
| 項目         | 用途                                        |
| ---------- | ----------------------------------------- |
| LAN（同網段）   | 透過 **MAC Address** 傳送 Ethernet Frame      |
| IP Address | 決定要到哪一台設備（邏輯位址）                           |
| ARP        | 將 IP 解析成 MAC                              |
| Switch     | 根據 MAC Address 轉送資料                       |
| Router     | 根據 IP Address 決定下一跳路由，並重新封裝新的 MAC Address |


# WAN
WAN（Wide Area Network，廣域網路）重點是：
WAN 不是單純用 MAC Address 找到最終設備，而是主要靠 IP Address + Router 路由轉送。
跨網段時，Router 用 IP 判斷下一站；但每一段實際送出去時，都還是要轉成該段網路可用的第二層位址，例如 Ethernet 的 MAC Address。
變化流程如下:
你的電腦
↓  目的 IP：8.8.8.8
↓  目的 MAC：家裡 Router 的 MAC

家裡 Router
↓  目的 IP：8.8.8.8
↓  目的 MAC：下一台 ISP Router 的 MAC

ISP Router
↓  目的 IP：8.8.8.8
↓  目的 MAC：下一台 Router 的 MAC

最後一個 Router
↓  目的 IP：8.8.8.8
↓  目的 MAC：Google Server 的 MAC

| 項目     | 會不會變           |
| ------ | -------------- |
| 目的 IP  | 通常不變           |
| 目的 MAC | 每一跳都會變         |
| Router | 看 IP 決定下一跳     |
| Switch | 看 MAC 轉送 Frame |

# NAT
========================
NAT（Network Address Translation）
中文：網路位址轉換
========================

【NAT 是什麼？】

NAT 是 Router 的一項功能。

作用：
將 LAN 內部的 Private IP
轉換成 Router 的 Public IP，
讓 LAN 內的設備可以存取 Internet。

例如：

PC
192.168.1.10
      │
      ▼
Router（NAT）
192.168.1.10
      ↓
114.35.100.50（Public IP）
      │
      ▼
Internet

========================
Router 的主要功能
========================

1. Routing（路由）
- 負責跨網段傳送封包。
- 決定封包要送往哪個網路。

2. NAT（Network Address Translation）
- 將 Private IP 轉換成 Public IP。
- 讓多台 LAN 設備共用一個 Public IP 上網。

========================
Private IP 與 Public IP
========================

# Private IP（私有 IP）
- 使用於 LAN。
- 可自行設定。
- 不可直接在 Internet 使用。
- 不同家庭可以重複使用相同的 Private IP。

# Public IP（公開 IP）
- 使用於 Internet。
- 通常由 ISP（例如中華電信）分配給 Router。
- 全球唯一，不能自行任意設定。
- 由全球 IP 位址管理制度統一管理與分配。

========================
一句話記住
========================

# Router：
主要負責跨網段傳送封包（Routing）。

# NAT：
是 Router 常見的一項功能，
負責將 Private IP 轉換成 Public IP，
讓 LAN 可以存取 Internet。

# Public IP：

# DHCP
DHCP = Dynamic Host Configuration Protocol
中文常翻成：動態主機設定協定
它的主要功能是：
自動幫設備分配網路設定。
一種網路協定 Protocol
DHCP 是一種協定，
用來實現「自動發 IP 設定」這個功能。
DHCP Server 可以自動發給設備：

IP Address
Subnet Mask
Gateway
DNS Server
Lease Time

例如：

IP Address：192.168.1.100
Subnet Mask：255.255.255.0
Gateway：192.168.1.1
DNS：192.168.1.1
Lease Time：24 小時
常見 DHCP Server 位置：
家庭網路：路由器
公司網路：防火牆、伺服器、路由器
工業網路：工業路由器、管理型交換器、IPC、專用網路設備
最常見的是：
路由器內建 DHCP Server
當設備設定成：

自動取得 IP
DHCP Mode
並且：

開機
插上網路線
網卡重新啟用
租約到期或更新

設備就會主動向網路尋找 DHCP Server。

流程是：
Discover → Offer → Request → Ack

簡稱：
DORA
DORA 流程
1. Discover
設備廣播：有沒有 DHCP Server？
2. Offer
DHCP Server 回覆：我可以給你這組 IP
3. Request
設備回覆：我要使用這組 IP
4. Ack
DHCP Server 確認：可以，這組設定給你



常見模式：
1. Static IP 固定 IP
2. DHCP 自動取得 IP
3. DHCP Reservation 固定租約
4. DHCP Exclusion 排除範圍

工業現場重點：
固定設備不要放在 DHCP Pool 裡，
否則可能發生 IP 衝突。

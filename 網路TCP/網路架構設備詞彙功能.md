# TCP IP VS UDP IP
| 比較項目   | TCP/IP                                            | UDP/IP                                     |
| ------ | ------------------------------------------------- | ------------------------------------------ |
| 全名     | Transmission Control Protocol / Internet Protocol | User Datagram Protocol / Internet Protocol |
| 中文     | **傳輸控制協定 / 網際網路協定**                               | **使用者資料報協定 / 網際網路協定**                      |
| OSI層   | TCP：第4層、IP：第3層                                    | UDP：第4層、IP：第3層                             |
| 是否建立連線 | ✅ 需要（三向交握，Three-way Handshake）                    | ❌ 不需要                                      |
| 是否保證送達 | ✅ 保證                                              | ❌ 不保證                                      |
| 是否確認收到 | ✅ 有 ACK（Acknowledgement，確認）                       | ❌ 沒有                                       |
| 是否重送   | ✅ 會                                               | ❌ 不會                                       |
| 是否保證順序 | ✅ 保證                                              | ❌ 不保證                                      |
| 傳輸速度   | 較慢                                                | 較快                                         |
| 封包大小   | Header較大（至少20 Bytes）                              | Header較小（8 Bytes）                          |
| 適合用途   | 檔案、網頁、資料庫                                         | 即時影音、語音、遊戲                                 |

TPC 與 UDP 是兩種網路技術但相同的他們都靠IP傳遞封包，但根據特性我們可以知道TCP傳輸協定可靠UDP因為不需確認他快速。
IP到底是什麼?IP是Internet Protocol，網際網路協定
| 項目                 | TCP   | UDP  |
| ------------------ | ----- | ---- |
| 是不是不同技術？           | ✅ 是   | ✅ 是  |
| 是不是傳輸層協定？          | ✅ 是   | ✅ 是  |
| 是不是廣播技術？           | ❌ 不是  | ❌ 不是 |
| 能不能用來做廣播？          | ❌ 不行  | ✅ 可以 |
| 能不能做單播？            | ✅ 可以  | ✅ 可以 |
| 能不能做多播（Multicast）？ | ❌ 不適合 | ✅ 可以 |


# gateway閘道器
不同網段的時候一定要經過閘道器
它的工作是：
連接兩種不同的通訊介面或不同的通訊協定。
| Gateway 類型       | 中文    | 功能               |
| ---------------- | ----- | ---------------- |
| Default Gateway  | 預設閘道器 | 封包跨網段時的出口（IP 路由） |
| Protocol Gateway | 協定閘道器 | 不同通訊介面或不同協定之間的轉換 |
Gateway 是用來連接不同網路、不同通訊介面或不同通訊協定的設備或節點。
只有 Default Gateway（預設閘道器） 幾乎都是第 3 層

# Switch交換器 VS Hub集線器
| 比較項目                       | Hub                     | Switch                     |
| -------------------------- | ----------------------- | -------------------------- |
| **OSI層**                   | 第1層（Physical Layer，實體層） | 第2層（Data Link Layer，資料鏈結層） |
| **判斷MAC位址**                | ❌ 不會                    | ✅ 會                        |
| **是否有MAC Table**           | ❌ 沒有                    | ✅ 有（MAC Address Table）     |
| **是否學習MAC**                | ❌ 不會                    | ✅ 會（MAC Learning）          |
| **封包轉送方式**                 | 收到就全部送                  | 查MAC Table，只送目的Port        |
| **Unicast（單播）**            | 全部Port都收到               | 只有目的Port收到                 |
| **Broadcast（廣播）**          | 全部送                     | 全部送（必要時）                   |
| **Unknown Unicast（未知MAC）** | 全部送                     | 全部送（第一次找不到MAC時）            |
| **Collision Domain（碰撞網域）** | 全部Port共用一個              | 每個Port各自獨立                 |
| **Broadcast Domain（廣播網域）** | 一個                      | 一個（未切VLAN時）                |
| **頻寬（Bandwidth）**          | 全部設備共享                  | 每個Port獨立                   |
| **雙工模式（Duplex）**           | Half Duplex（半雙工）        | Full Duplex（全雙工）           |
| **碰撞（Collision）**          | 容易發生                    | 幾乎沒有                       |
| **傳輸效率**                   | 低                       | 高                          |
| **安全性**                    | 差（所有設備都收到封包）            | 較佳（只有目的Port收到）             |
| **價格**                     | 較便宜（現在已很少使用）            | 較貴（目前主流）                   |
| **目前是否常見**                 | 幾乎淘汰                    | 幾乎所有企業與家庭都使用               |

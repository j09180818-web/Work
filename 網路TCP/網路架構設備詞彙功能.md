連接兩種不同的通訊介面或不同的通訊協定。
| Gateway 類型       | 中文    | 功能               |
| ---------------- | ----- | ---------------- |
| Default Gateway  | 預設閘道器 | 封包跨網段時的出口（IP 路由） |

Gateway 是用來連接不同網路、不同通訊介面或不同通訊協定的設備或節點。
只有 Default Gateway（預設閘道器） 幾乎都是第 3 層

# Subnet子網路與 Subnet Mask（子網路遮罩）
子網路遮罩是一種規則，他告訴電腦IP哪部分代表子網路而哪部分代表主機
IP
192.168.1.100

Mask
255.255.255.0

192.168.1 | 100
──────────┬────
 子網路      主機
因為它像一個模板，把 Host（主機） 的部分遮住，只留下 Network（子網路）。
Subnet Mask（子網路遮罩）本質上就是一個二進位（Binary）的遮罩。
255=11111111二進制所以整排就是11111111.11111111.11111111.00000000
如何遮罩呢
跟IP做AND運算這樣得到的結果當後面都是0的時候所有號碼都是0只保持前面為1的數值那就可以知道前面數字為網段

另外說明為什麼Subent Mask都是特定幾個數字因為呢左邊一定是連續的右邊連續零故只有下列幾種數字
| 二進位      | 十進位 |
| -------- | --- |
| 11111111 | 255 |
| 11111110 | 254 |
| 11111100 | 252 |
| 11111000 | 248 |
| 11110000 | 240 |
| 11100000 | 224 |
| 11000000 | 192 |
| 10000000 | 128 |
| 00000000 | 0   |

CIDR是什麼呢無類別域間路由表示有幾個1標記方法/number
| CIDR | 子網路遮罩 Subnet Mask |
| ---- | ----------------- |
| /24  | 255.255.255.0     |
| /25  | 255.255.255.128   |
| /26  | 255.255.255.192   |
| /27  | 255.255.255.224   |
| /28  | 255.255.255.240   |
| /29  | 255.255.255.248   |
| /30  | 255.255.255.252   |---->代表最後有兩個Host Bit共四個IP但
| IP          | 用途                     |
| ----------- | ---------------------- |
| 192.168.1.0 | Network Address，網路位址   |
| 192.168.1.1 | Host，可用                |
| 192.168.1.2 | Host，可用                |
| 192.168.1.3 | Broadcast Address，廣播位址 |

IPv4 總共 32 bit而由此可以計算出我們有幾個Bit可以用
可用 Host 數 = 2^n - 2其中n = Host Bit（Host 位元數）
-2的原因是因為以下兩個位子必須占用
在一般 IPv4 子網路中：
第一個 IP（最小的 IP） → Network Address（網路位址）
最後一個 IP（最大的 IP） → Broadcast Address（廣播位址）
中間所有 IP → 可分配給 Host 使用

1. 子網路遮罩中，被遮住的地方是網段位。
   例如 255.255.0.0，代表前兩組是網段號碼 X.X。

2. 沒被遮住的地方是 Host 位。
   例如 192.168.0.0/16，可用 Host 是 192.168.0.1 ~ 192.168.255.254。

3. 如果遮罩出現 128、192、224、240、248、252，
   代表遮罩切在某一組中間，需要算跳號。

4. 跳號公式：
   跳號 = 256 - 遮罩數字

5. 每一個網段的第一個 IP 是 Network Address，
   每一個網段的最後一個 IP 是 Broadcast Address，
   中間才是可用 Host。

   # 子網路遮罩計算小筆記

## 1. 子網路遮罩的作用

子網路遮罩 Subnet Mask 用來決定：

```text
哪一部分是網段 Network
哪一部分是主機 Host
```

簡單記：

```text
255 的地方 = 網段位，固定
0 的地方 = Host 位，可變
128 / 192 / 224 / 240 / 248 / 252 = 遮到一半，要算跳號
```

---

## 2. 常見遮罩對照

| 子網路遮罩           | CIDR | 網段位       | Host 位    |  每段可用 Host |
| --------------- | ---: | --------- | --------- | ---------: |
| 255.0.0.0       |   /8 | 第 1 組     | 第 2、3、4 組 | 16,777,214 |
| 255.255.0.0     |  /16 | 第 1、2 組   | 第 3、4 組   |     65,534 |
| 255.255.255.0   |  /24 | 第 1、2、3 組 | 第 4 組     |        254 |
| 255.255.255.128 |  /25 | 前 25 bit  | 後 7 bit   |        126 |
| 255.255.255.192 |  /26 | 前 26 bit  | 後 6 bit   |         62 |

---

## 3. 如果遮罩切到一半，要算跳號

只要看到：

```text
128、192、224、240、248、252
```

就要算跳號。

公式：

```text
跳號 = 256 - 遮罩數字
```

範例：

```text
255.255.255.192
跳號 = 256 - 192 = 64
```

所以最後一組會切成：

```text
0 ~ 63
64 ~ 127
128 ~ 191
192 ~ 255
```

---

## 4. 每一個網段的規則

每一個網段都有：

```text
第一個 IP = Network Address 網路位址，不能給設備用
中間 IP = 可用 Host IP
最後一個 IP = Broadcast Address 廣播位址，不能給設備用
```

---

## 5. 例題

題目：

```text
IP：192.168.10.70
Mask：255.255.255.192
```

### 步驟 1：算 CIDR

```text
255.255.255.192 = /26
```

因為：

```text
255 = 8 bit
255 = 8 bit
255 = 8 bit
192 = 2 bit

8 + 8 + 8 + 2 = 26
```

### 步驟 2：算跳號

```text
跳號 = 256 - 192
跳號 = 64
```

### 步驟 3：列出網段

```text
第 1 段：192.168.10.0   ~ 192.168.10.63
第 2 段：192.168.10.64  ~ 192.168.10.127
第 3 段：192.168.10.128 ~ 192.168.10.191
第 4 段：192.168.10.192 ~ 192.168.10.255
```

### 步驟 4：判斷 IP 落在哪段

```text
IP：192.168.10.70

看最後一組 70
70 落在 64 ~ 127
所以它在第 2 段
```

---

## 6. 例題答案

```text
IP：192.168.10.70
Mask：255.255.255.192
CIDR：/26
跳號：64

所屬網段：
192.168.10.64 ~ 192.168.10.127

Network Address：
192.168.10.64

可用 Host：
192.168.10.65 ~ 192.168.10.126

Broadcast Address：
192.168.10.127
```

---

## 7. 最重要記法

```text
先看 Mask
再算跳號
再找 IP 落在哪一段

每段第一個是 Network
每段最後一個是 Broadcast
中間才是 Host
```

一句話：

```text
遮罩越多，網段切越細，每個網段可用 Host 越少。
```

# MAC :MAC 層
媒體存取控制層
在 TCP/IP 或 OSI 架構中，MAC 層屬於：
OSI 第 2 層：資料鏈結層 

資料鏈結層 Data Link Layer又可以再細分成兩個部分：
├─ LLC 層：Logical Link Control
└─ MAC 層：Media Access Control
在同一個 LAN 裡，資料要送給哪一台設備，以及大家如何輪流使用網路線或無線電波。
此層功能有
1.指定區域網路內的收件人
IP 位址通常用來判斷「目標在哪個網路」。
MAC 位址則是用來判斷「在這個區域網路內，資料要交給哪一台設備」。
在傳遞網路資料時封包會如下
Ethernet Frame乙太網路
├─ 目的 MAC：PLC 的 MAC Address
├─ 來源 MAC：HMI 的 MAC Address
├─ 資料內容：IP Packet / TCP / Modbus TCP
└─ 檢查碼

而在LAN中主要看的就是MAC ADDRESS
在以前使用 Hub 或半雙工網路時，多台設備共用同一條傳輸媒介，可能會發生碰撞。
MAC層就會決定什麼時候送如果碰撞後如何解解決
不過現代 Switch 網路多半是全雙工 Full-Duplex，碰撞問題已經很少見。
1. MAC 層本身不負責管 IP。
2. MAC 層負責 LAN 內 Ethernet Frame 的來源 MAC / 目的 MAC。
3. IP 層才負責來源 IP / 目的 IP / 子網路 / Gateway / 路由。
4. 同網段時，目的 MAC 是目標設備。
5. 跨網段時，目的 MAC 是 Gateway。
6. IP 目的地可以是遠端設備，但 MAC 目的地永遠只是目前這一段 LAN 的下一站。
# MAC ADDRESS
MAC Address / 實體位址 / 硬體位址 是網路卡的硬體識別碼
MAC Address 通常是 48 bits，也就是 6 Bytes。
00-1A-2B-3C-4D-5E
拆分成
00-1A-2B    -    3C-4D-5E
廠商代碼          廠商自行分配編號
OUI              NIC Specific
1. MAC Address 理論上設計為全球唯一。
2. MAC Address 通常由廠商 OUI + 廠商分配編號組成。
3. 48-bit MAC Address 數量非常大。
4. 正常出廠設備不應該重複。
5. 但人為修改、VM 複製、廠商燒錄錯誤，可能造成重複。
6. 只要在同一個 LAN / VLAN 內 MAC 重複，就可能造成通訊異常。
7. MAC 重複常見現象是 Switch MAC Table 跳動，也就是 MAC Flapping。

中文常叫：

這台設備在哪個網路？

| ---- | ----------------------- | ------------------------- |
| 中文   | IP 位址 / 邏輯位址            | 實體位址 / 硬體位址               |

| 常見格式 | 192.168.1.10            | 00-1A-2B-3C-4D-5E         |

因此總結

# Switch交換器 VS Hub集線器
| 比較項目                       | Hub                     | Switch                     |
| -------------------------- | ----------------------- | -------------------------- |
| **OSI層**                   | 第1層（Physical Layer，實體層） | 第2層（Data Link Layer，資料鏈結層） |
| **判斷MAC位址**                | ❌ 不會                    | ✅ 會                        |
| **是否有MAC Table**           | ❌ 沒有                    | ✅ 有（MAC Address Table）     |
| **是否學習MAC**                | ❌ 不會                    | ✅ 會（MAC Learning）          |
| **封包轉送方式**                 | 收到就全部送                  | 查MAC Table，只送目的Port        |
| **Unicast（單播）**            | 全部Port都收到               | 只有目的Port收到                 |
| **Broadcast（廣播）**          | 全部送                     | 全部送（必要時）                   |
| **Unknown Unicast（未知MAC）** | 全部送                     | 全部送（第一次找不到MAC時）            |
| **Collision Domain（碰撞網域）** | 全部Port共用一個              | 每個Port各自獨立                 |
| **Broadcast Domain（廣播網域）** | 一個                      | 一個（未切VLAN時）                |
| **頻寬（Bandwidth）**          | 全部設備共享                  | 每個Port獨立                   |
| **雙工模式（Duplex）**           | Half Duplex（半雙工）        | Full Duplex（全雙工）           |
| **碰撞（Collision）**          | 容易發生                    | 幾乎沒有                       |
| **傳輸效率**                   | 低                       | 高                          |
| **安全性**                    | 差（所有設備都收到封包）            | 較佳（只有目的Port收到）             |
| **價格**                     | 較便宜（現在已很少使用）            | 較貴（目前主流）                   |
| **目前是否常見**                 | 幾乎淘汰                    | 幾乎所有企業與家庭都使用               |
